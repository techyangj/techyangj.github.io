Shell脚本

提前设计可执行语句，用来完成特定任务的文件
解释型程序
顺序、批量执行
常见的脚本语言：
bash shell
python/perl/puby
JSP/PHP/ASP/CGL
Javascript
shell语言的后缀名以.sh结尾
注意写完脚本要赋予权限：chmod +x

脚本组成

1、#！环境声明（以下的所有代码以哪种语言执行）
python–>>#!usr/bin/python
shell–>> #!/bin/bash
2、#注释文本
3、可执行代码

重定向输出

> 正确输出
2> 错误输出
&> 正确输出和错误输出都进行

反撇号’’与$():将命令的输出，当作参数直接参与下一个命令的运行

变量

变量：为了适应多变的环境需求
变量：会变化的值
变量名=存储的值
使用变量存储的值：$变量名

变量的意义：
方便以固定名称重复使用某个值
提高对任务需求，运行环境变化的适应能力

变量的注意事项：
若指定的变量名已存在，相对于为此变量重新赋值
等号两边不要有空格
变量名由下划线、字母和空格组成，且区分大小写
变量名不能以数字开头、不能使用关键字和特殊字符

基本格式：
引用变量名：$变量名
查看变量名：echo $变量名 echo ${变量名}

为了用户的体验，降低脚本的使用难度，产生交互
向脚本中传递命令行参数
read -p ‘屏幕输出信息’
1、产生交互
2、记录用户在键盘上的值
3、将记录的信息赋值给一个变量储存

案例：创建用户并且设置密码
黑洞设备：/dev/null

变量区分

类型 说明
环境变量 变量名一般为大写，用来设置用户、系统环境
位置变量 bash内置，存储执行脚本时提供命令行参数
预定义变量 bash内置，可直接调用的特殊值，不能直接修改
自定义变量 用户自主设置、修改及使用

  • 环境变量
    常见的环境变量：
    PWD、PATH、USER、LOGNAME、SHELL、HOME

  • 位置变量
    直接使用这些变量，且不能直接为这些变量赋值
    类似$1,$2之类的位置变量

变量名 含义
$# 已加载的位置变量的个数
$? 程序退出后的状态，0表示正常，其他值表示异常
  • 预定义变量
    用来保存脚本程序的执行信息
    直接使用这些

常用的测试选项

  • 检查文件状态
    -e 文档存在才为真
    -d 文档存在，且必须为目录才为真
    -f 文档存在，且必须为文件才为真

  • 比较整数大小
    -gt 大于
    -eq 等于
    -ne 不等于
    -lt 小于
    -le 小于等于
    -ge 大于等于

  • 字符串对比
    == : 等于 != ：不等于
